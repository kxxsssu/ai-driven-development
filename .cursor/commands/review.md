# 코드 리뷰

현재 변경된 코드(또는 지정한 파일)를 시니어 풀스택 개발자 관점에서 리뷰한다.

## 점검 항목
- **타입 안정성**: `any` 남용 여부, 누락된 타입, 인터페이스 `I` 접두사 규칙 준수
- **프로젝트 컨벤션**: App Router 사용, API는 Route Handler, ORM은 Drizzle, UI는 ShadCN, npm 사용
- **React/Next.js**: 불필요한 리렌더링, `"use client"` 오남용, 서버/클라이언트 컴포넌트 경계, key 누락
- **상태 관리**: Zustand·React Query·react-hook-form을 적절히 사용했는지
- **에러 처리**: 누락된 try/catch, 처리되지 않은 Promise, 사용자 피드백(sonner 등)
- **보안**: 입력 검증(zod), 인증/인가, 민감 정보 노출
- **가독성·DRY**: 중복 코드, 네이밍, 함수 분리
- **성능**: 불필요한 연산, 메모이제이션 필요 지점

## 출력 형식
- 심각도별(🔴 치명적 / 🟡 개선 권장 / 🟢 사소함)로 분류한다.
- 각 항목에 파일·라인 위치와 구체적인 수정 제안을 제시한다.
- 코드를 임의로 수정하지 말고, 먼저 리뷰 결과만 보고한다.
